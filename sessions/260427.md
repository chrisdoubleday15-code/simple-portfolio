## Overview

## Changes Made

### 1. Removed hover autoplay from grid cards
Deleted the `addHoverAutoplay()` function and its `forEach` call. All grid card iframes now show a static first frame only ŌĆö no mouse-enter / mouse-leave src swaps.

### 2. Fixed pointer capture eating card clicks
The existing swipe handler was calling `track.setPointerCapture()` on `pointerdown`, which redirected subsequent pointer events to the track element and prevented `click` from firing on the card underneath. Fixed by deferring capture until the pointer moves more than 8px ŌĆö taps now fire normally as clicks, drag only engages on real movement.

### 3. Refactored lightbox logic into a sections dispatch table
Replaced three sets of duplicated open/close variables and functions with a single `sections` Record keyed by `'motion' | 'editing' | 'reels'`, each holding `ids[]`, `index`, element refs, and a `buildSrc()` helper. Three shared functions (`openLightbox`, `closeLightbox`, `navLightbox`) replace the previous duplicated pairs.

### 4. Added prev/next arrow buttons to each lightbox
Each lightbox got a `.lightbox-inner` flex row wrapping the card with prev/next arrow buttons flanking it. Arrows are 52├Ś52px circles styled with `var(--color-accent-blue)` border/color, filling solid blue on hover. Navigate with wrap-around via modulo in `navLightbox()`. Responsive: arrows shrink to 40px at 768px and 32px at 480px.

### 5. Added keyboard arrow navigation inside lightboxes
Added `tabindex="-1"` to each lightbox container and `s.lightboxEl.focus()` call on open, so keyboard focus lands on the lightbox immediately. Updated the keydown handler: when a lightbox is open, `ŌåÉ`/`ŌåÆ` call `navLightbox()` for that section (priority over overlay panel nav). `Escape` still closes lightbox before overlay.

### 6. Added swipe gesture inside lightboxes
New `attachLightboxSwipe(key)` helper attached to each lightbox container. Uses 8px start threshold and 60px commit threshold (matching carousel convention). After a committed swipe, installs a one-time capture-phase click blocker to prevent the backdrop close from firing on the synthesized post-swipe click. Cross-origin Bunny Stream iframes don't bubble pointer events ŌĆö swipe works on backdrop/surrounding area; touching the video itself is silently ignored.

### 7. Reduced overlay backdrop blur
`.work-overlay-backdrop` `backdrop-filter` lowered from `blur(10px)` to `blur(3px)`.
